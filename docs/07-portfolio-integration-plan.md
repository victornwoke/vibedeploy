# SECTION 7 -- PORTFOLIO INTEGRATION PLAN

## 7.1 Strategy

VibeDeploy strengthens victornwoke.com by adding a live, working product to the
portfolio -- not just static project cards. The integration must feel natural
within the existing design language (dark purple/blue theme, rounded cards,
badge system) and must not disrupt the primary portfolio user experience.

victornwoke.com remains the primary domain and professional anchor.
vibedeploy.victornwoke.com is the product. victornwoke.com/projects/vibedeploy
is the case study.

## 7.2 Step-by-Step Integration

### Step 1 -- Add "Featured Product" Section to Portfolio Homepage

Add a new section between "Featured Projects" and "Professional Experience" on
victornwoke.com. The section headline: "Live Product -- VibeDeploy". Include a
card styled to match the existing portfolio card system with:

- VibeDeploy logo or product name in purple gradient text

- One-line description: "Production readiness checker for AI-built apps"

- Technology badge row: React, TypeScript, Tailwind, GitHub Actions,
  Azure/Vercel

- "Live App" button → vibedeploy.victornwoke.com

- "Case Study" button → victornwoke.com/projects/vibedeploy

- FEATURED badge in top-right corner of card

### Step 2 -- Add VibeDeploy to Project Cards Grid

Add VibeDeploy as the first card in the Featured Projects grid on
victornwoke.com. The card should include the production domain, GitHub
repository link, and a screenshot or mockup of the report results page as the
card image.

### Step 3 -- Create the Case Study Page at /projects/vibedeploy

The case study page demonstrates technical depth to enterprise recruiters.
Include:

- Problem: Why I built VibeDeploy -- the AI-app deployment gap

- Architecture: Tech stack diagram and folder structure walkthrough

- Scoring Engine: How the 13-category model was designed and weighted

- CI/CD: The GitHub Actions pipeline explained

- Deployment: Azure Static Web Apps and/or Vercel setup

- Lessons Learned: What would change in Version 2

- Live Demo link → vibedeploy.victornwoke.com

- GitHub link → repository

### Step 4 -- VibeDeploy Footer → victornwoke.com

VibeDeploy's footer must include: "Built by Victor Nwoke -- Cloud & DevOps
Engineer" with a clickable link to victornwoke.com. This creates reciprocal
navigation and signals to recruiters that both properties are owned by the same
expert.

### Step 5 -- Founder Proof Section on VibeDeploy

The VibeDeploy About or Founder section links prominently to victornwoke.com as
the primary professional profile. It should not duplicate full portfolio content
but should reinforce trust through key credentials and a clear "See Full
Profile" link.

## 7.3 What NOT to Do

- Do NOT move victornwoke.com to portfolio.victornwoke.com -- this breaks
  existing SEO and links

- Do NOT make VibeDeploy the homepage of victornwoke.com -- keep them separate
  products

- Do NOT use a different design language on VibeDeploy that breaks brand
  continuity

- Do NOT duplicate the full portfolio content on the VibeDeploy About page
